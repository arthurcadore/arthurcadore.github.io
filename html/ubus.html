UBUS (MICRO BUS)

ubus (OpenWrt micro bus architecture)

Para fornecer comunicação entre processos entre vários daemons e aplicativos instalados no OpenWrt (distribuição linux instalada nos APs, voltada para a wireless), um projeto chamado “ubus” foi desenvolvido. Ele consiste em várias partes, incluindo um daemon próprio para atuar em segundo plano, biblioteca e alguns auxiliares extras. O coração deste projeto é o “ubusd” daemon. Ele fornece uma interface para que outros daemons se registrem e enviem mensagens. Para os curiosos, essa interface é implementada usando soquetes Unix e usa mensagens TLV (tipe-length-value) . Cada daemon registra um conjunto de caminhos em um namespace específico. Cada caminho pode fornecer vários procedimentos com qualquer número de argumentos. Os procedimentos podem responder com uma mensagem. A “ubus” ferramenta de linha de comando permite interagir com o deamon “ubusd” (com todos os serviços registrados atualmente). É útil para investigar/depurar configurações, bem como escrever scripts de shell. Para chamar procedimentos com parâmetros e retornar respostas, ele usa o formato JSON.

ubus list copiar

O ubus list é utilizado para verificar as chamadas ubus disponíveis para execução, cada chamada terá um conjunto de parâmetros individuais (métodos), e então este poderá ter vários parâmetros diferentes para cada método. Algumas das principais chamadas são:

system: lista as principais informações do sistema, como versão de firmware, memória, CPU, entre outros.

iwinfo: lista as informações da wireless do dispositivo, como os clientes conectados e status de cada interface wireless.

uci: lista as configurações do dispositivo, as configurações são exibidas no formato “json”, onde cada parâmetro tem um valor correspondente.

itb.system.config: lista as possíveis operações com o arquivo de configuração do sistema (backup ou reset), porém, via SSH apenas o reset está disponível.

itb.service.cronos: lista as informações enviadas pelo server cronos.

itb.service.facebookwifi: lista as informações de conexão entre o facebook e o AP (para a operação do checkin facebook).

network.device: lista o status da interface selecionada, com todos os valores de conexão
